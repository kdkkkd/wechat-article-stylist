# Layout Recipes

## 用法

把这些模块当作可复制的排版积木，根据文章气质选 2 到 4 个组合使用。优先统一风格，不要整篇混搭。

## 开头模块

### 直接提问型

```text
你有没有发现，
[读者常见困惑]

很多人以为 [常见误解]，
但真正拉开差距的，往往是 [核心观点]。
```

### 反差切入型

```text
同样是 [同一场景]，
有人越做越顺，
有人却一直卡在原地。

差别不一定在努力，
而在于有没有看清 [关键因素]。
```

### 结论前置型

```text
先说结论：
[一句话核心判断]

如果你正在面临 [场景]，
这件事值得你现在就重视。
```

## 小标题模块

### 编号型

```text
01 先看清问题
02 再找到关键
03 最后开始行动
```

### 情绪型

```text
先别急着否定自己
真正该改变的，可能不是努力
把节奏找回来，比硬撑更重要
```

## 强调模块

### 金句单列

```text
[一句适合单独成段的核心表达]
```

### 提醒框

```text
值得提醒的是：
[一个容易被忽略但很关键的点]
```

### 引用感

```text
不要急着追求面面俱到，
先把最重要的一件事做对。
```

## 清单模块

### 三点法

```text
你可以先从三件事开始：

1. [动作一]
2. [动作二]
3. [动作三]
```

### 对照法

```text
不是 [低效做法]，
而是 [更优做法]。

不是 [表面问题]，
而是 [真正问题]。
```

## 结尾模块

### 收束总结型

```text
说到底，
[主题] 从来不是 [误区]，
而是 [真正结论]。

当你开始 [关键动作]，
很多问题就会慢慢变得清晰。
```

### 互动引导型

```text
你最近在 [主题] 上遇到过什么问题？
欢迎留言聊聊。

如果这篇内容对你有启发，
也欢迎转发给那个正需要的人。
```

## 排版提醒

- 同一篇文章里，分隔模块不要超过 3 种。
- 如果内容偏专业，少用煽情句，多用结论句。
- 如果内容偏情绪，减少硬编号，增加停顿和共鸣句。
- 手机端第一屏很重要，开头不要塞太多背景资料。
